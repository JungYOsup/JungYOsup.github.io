## 1. float(1)

- **float를 사용하는이유**: block 요소는 자기 다음의 오는 요소들이 자기를 침범하지 않게 하는데, 그러면 가로배치가 굉장히 어렵게 된다.
  이를 해결하기 위해서 float를 사용한다.(즉 블록요소의 가로배치를 하기 위해서)

    - inline , inline block, block ---> block 이 되지만 다음과 같은 요소가 **반영 안됨**

      - 박스에 width를 선언하지 않을 경우 , **width = 부모의 content-box의 100%** 이지만 float가 선언된 박스는 자기 자신의 콘텐츠 크기만큼 width를 가지게 된다.

## 2. float(2)

- **float로 인해서 변하는 레이아웃들을 어떻게 잡을지에 대해서 알아보자**

  - 1.overflow:hidden 을 사용한다.(**별로 권장하지 않는방법**)
    - 원래는 부모에게 hegiht를 주지 않고 자식요소 모두에 float를 줄경우 부모의 hegiht는 0px 이 된다고 했는, 부모에게 **overflow : hidden을 하게 되면 집나간 자식들을 부모가 찾는다.**

  * 2.clear 을 사용한다.

    - 집나간 자식요소를 다른 자식이 인식하게 된다.

  ![7](/assets/images/css_img/7.PNG)

  ```Css
  * {
  box-sizing: border-box;
  }
  parent {
  width: 200px;
  margin: 0 auto;
  background-color: gray;
  }
  .child {
  width: 200px;
  height: 200px;
  font-size: 5px;
  font-weight: bold;
  text-align: center;
  color: white;
  }
  .red {
  background-color: red;
  }
  .blue {
  background-color: blue;
  }
  ```

  ![8](/assets/images/css_img/8.PNG)

  ```Css
  * {
  box-sizing: border-box;
  }
  parent {
  width: 200px;
  margin: 0 auto;
  background-color: gray;
  }
  .child {
  width: 200px;
  height: 200px;
  font-size: 5px;
  font-weight: bold;
  text-align: center;
  color: white;
  }
  .red {
  background-color: red;
  float : left
  }
  .blue {
  background-color: blue;
  }
  ```

  ![9](/assets/images/css_img/9.PNG)

  ```Css
  * {
  box-sizing: border-box;
  }
  parent {
  width: 200px;
  margin: 0 auto;

  }
   .child {
  width: 200px;
  height: 200px;
  font-size: 5px;

  color: white;
  }
   .red {
  background-color: red;
  float : left
  }
  .blue {
  background-color: blue;
  clear : left
  }
  ```
